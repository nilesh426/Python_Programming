## About This Repository

This repository is a step-by-step Python learning journey organized by days. Each day focuses on a specific set of topics so learners can build fundamentals gradually through small, easy-to-understand programs.

It is useful for:

- Beginners starting Python from scratch
- Students practicing for assignments, exams, or mini-projects
- Developers revising Python fundamentals
- Anyone who wants to learn by writing and running code

The main goal of this repository is learning through practice, not just reading theory.

## Requirements

Before you begin, make sure you have:

- **Python 3.8 or newer**
- **Terminal or command-line access**
- **A code editor**, such as VS Code, PyCharm, or any editor that supports Python files
- **Basic computer knowledge**, such as opening folders and running commands

Check your Python version with:

```bash
python --version
```

If `python` does not work on your system, try:

```bash
py --version
```

## How to Run the Code

1. Open your terminal or command prompt.
2. Move into the repository folder.
3. Move into the day folder you want to practice.
4. Run the Python file.

Example:

```bash
cd course_prashant_jha/DAY2
python List.py
```

On Windows, you can also use:

```bash
py List.py
```

For interactive scripts, follow the prompts shown in the terminal.

If something does not work, check that:

- Python is installed properly
- You are in the correct folder
- The file name is typed correctly
- The file extension is `.py`

## Suggested Learning Path

1. Start with `DAY1` to build your Python basics.
2. Continue to `DAY2` and `DAY3` for loops, strings, functions, and dictionaries.
3. Use `DAY4` to learn practical concepts like files and exceptions.
4. Move to `DAY5` and `DAY6` for object-oriented programming and stack-based thinking.
5. Practice `DAY7` for linked structures and efficiency.
6. Explore `DAY8`, `DAY9`, and `DAY10` for regex, recursion, trees, and graphs.
7. Use `Problem_solving` for extra practice after learning the basics.

If a topic feels difficult, rerun the examples, change values, and write your own version of the same problem.

## Tips for Learners

- Read the comments inside the scripts.
- Change input values and run the code again.
- Debug slowly when you get errors.
- Write your own version of each problem.
- Practice daily, even if only for a short time.
- Use the official Python documentation when you want deeper explanations.
